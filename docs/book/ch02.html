chapter 02

values and bindings

a binding is a name, one space, =, one space, a value. a block-form main indents its body by two spaces and each line binds a name — except the last, which is the value of the whole thing. that's the complete ceremony:

order.kso
main =
  age = 47
  price = 19.99
  name = "clay"
  print "{name} is {age}, owes {price}"
output
clay is 47, owes 19.99

three kinds of value so far. int is arbitrary-precision — it does not overflow, ever, and chapter 03 will compute 20! to prove it. float64 is the ieee double you know. strings interpolate with {braces}, and the braces take any expression, not just a name:

interp.kso
main =
  guests = 4
  print "table for {guests + 1}, please"
output
table for 5, please

numbers don't blur

ints and floats are different types, and kanso will not quietly promote one into the other. mixed arithmetic is an error with the fix in the message:

mixed.kso
main =
  total = 3 + 0.5
  print "{total}"
kanso run mixed.kso
error[runtime]: no implicit numeric coercion; convert explicitly with `to_float`
  --> mixed.kso:2:13
   2 |   total = 3 + 0.5
                   ^

do what it says and everything is fine — and notice the float renders as 3.5, never a bare 3 pretending to be an int. floats always show their point:

conv.kso
main =
  scoops = 3
  total = to_float scoops + 0.5
  print "{scoops} scoops plus a taster: {total}"
output
3 scoops plus a taster: 3.5

constants

a top-level binding is a constant, and main is simply the constant the runtime asks for. constants sit in alphabetical order with everything else in the file — yes, enforced; we'll get to that:

tau.kso
main = print "tau is {tau}"

tau = 6.28318
output
tau is 6.28318

rebinding

values are immutable — nothing in kanso mutates, ever. but a name may be rebound to a new value, which reads naturally when a value evolves through steps:

tab.kso
main =
  total = 10
  total = total + 5
  print "total: {total}"
output
total: 15

with one condition: each version of the name must actually be used before the next one replaces it. rebind over a value nobody read, and kanso asks why the first line exists at all:

rebind.kso
main =
  x = 1
  x = 2
  print "{x}"
kanso check rebind.kso
error[unused]: unused binding: each version of `x` is used before the next
  --> rebind.kso:2:3
   2 |   x = 1
         ^

nothing wasted

that error is not an outlier; it is the personality of the language. kanso's founding principle says anything a style guide or code review would enforce by convention becomes a compile error — which means the compiler treats waste the way other compilers treat type mismatches. a binding nobody reads:

leftover.kso
main =
  brand = "kanso"
  print "hello"
kanso check leftover.kso
error[unused]: unused binding `brand`
  --> leftover.kso:2:3
   2 |   brand = "kanso"
         ^

an expression that computes something and drops it on the floor:

drift.kso
main =
  1 + 1
  print "hello"
kanso check drift.kso
error[unused]: unused expression: every non-final line binds a name (sequence effects with `>>`)
  --> drift.kso:2:5
   2 |   1 + 1
           ^

whitespace you can't even see:

invisible.kso
main = print "hi" 
kanso check invisible.kso
error[formatting]: trailing whitespace is not part of the canonical grammar
  --> invisible.kso:1:18
   1 | main = print "hi"
                        ^

and the muscle memory you brought from every other language:

commas.kso
fn add a, b
  a + b

main = print "{add 1 2}"
kanso check commas.kso
error[formatting]: kanso has no commas; enumerations are space-separated
  --> commas.kso:1:9
   1 | fn add a, b
               ^

there are no commas. anywhere. arguments, list elements, record fields — everything is space-separated, and after a day it is hard to remember why other languages spend a keystroke between every pair of things.

read those diagnostics again and notice their posture: each states the rule as if it had always been obvious, because in this grammar it has. none of this is a linter you can disable or a formatter you run before review. there is no formatter — canonical form is the grammar, so a second tool would have nothing to do. the practical consequence: every diff you ever review in kanso is purely semantic. no reflowed lines, no whitespace churn, no "style nit" comments. the source contains only decisions.

ensō-neko

exactly one space around =, two-space indents, declarations in alphabetical order, nothing trailing. it isn't a style i prefer; it's the only rendering that parses. i sleep well because you cannot wake me.

mugi

don't fear the red text — i deliver it in ten milliseconds, fix included. type the mistake on purpose once. cheapest lesson in the language.
