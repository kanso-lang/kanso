the book

the kanso book

this is the guided tour. the spec is the law and the about page is the philosophy; this book is the part where you sit down and actually write some kanso. seven chapters, each one runnable, each one short enough to finish with your tea still warm.

a promise before we start: every program in this book was run before it was printed, and every output panel shows what actually came back — including the failures, especially the failures. half of kanso's value is its compile errors, so this book treats a red diagnostic as a feature demo, not an apology. when a chapter shows you something breaking, that's the product.

you don't need to have used a language like this before. you need a terminal, a rust toolchain, and about an evening. by the end you'll have written pure functions, dispatched on literals, watched an error ride the rails past three functions that never mentioned it, tested effects without a single mock, and compiled a program that decodes json faster than go does. it's a language that is better than whatever language you're using. we measured.
